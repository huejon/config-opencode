# Skill Builder

## Procedure

1. Name the repeated failure or recurring workflow.
2. Check whether an existing skill, command, or memory already covers it.
3. Define the exact trigger.
4. Write a short procedure and stop rules.
5. Add an output contract and failure modes.
6. Test in a sandbox.
7. Record rollback/removal path.

## Reject

Reject generic wisdom, project dumps, giant prompts, stale model folklore, and permission rules disguised as skills.
